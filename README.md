# Інформаційна Система «Квітковий Магазин»
## Архітектурна структура та технічний опис проекту

Цей документ містить опис структури каталогів, архітектурних рішень, взаємозв'язків між класами та призначення файлів в інформаційній системі квіткового магазину.

---

## Огляд архітектури

Проект побудовано відповідно до архітектурного шаблону MVC (Model-View-Controller) із рівнем збереження даних (Persistence Layer) у базі даних SQLite:

*   **Model (Моделі)**: класи пакету `flowers` (квіти, аксесуари, букет), що містять бізнес-дані та базові алгоритми (розрахунок вартості, сортування, пошук).
*   **View (Представлення)**: панелі інтерфейсу користувача в пакеті `ui` (`CatalogPane`, `BouquetPane`, `SearchPane`), стилізовані за допомогою CSS (`styles.css`).
*   **Controller / Service (Контролери / Сервіси)**: клас `FlowerShop.java`, який керує бізнес-логікою та взаємодіє з базою даних через `DatabaseManager.java`.
*   **Persistence Layer (База даних)**: клас `DatabaseManager.java`, який відповідає за підключення до SQLite та виконання SQL-запитів.

## Опис пакетів та файлів

### Точка входу
*   **`src/Main.java`**: Клас-запускник програми. Ініціалізує процес логування, фіксує запуск додатку та викликає системний метод `Application.launch(FlowerShopApp.class)` для передачі контролю графічному середовищу JavaFX.

### Пакет `src/flowers/` (Бізнес-моделі)
Містить об'єктні моделі квіткового асортименту, аксесуарів та букетів із використанням принципів успадкування, інкапсуляції та поліморфізму.

*   **`Flower.java`**: Абстрактний суперклас для всіх квітів. Містить поля: `id`, `name`, `price`, `stemLength`, `color` та `freshnessDate`. Оголошує абстрактний метод `getTypeName()`, який кожен нащадок реалізує для отримання назви виду квітки.
*   **`Rose.java`**: Підклас Троянди. Містить специфічні поля: `hasThorns` (наявність шипів), `budShape` (форма бутона).
*   **`Tulip.java`**: Підклас Тюльпана. Містить специфічні поля: `petalShape` (форма пелюсток), `isDouble` (чи є квітка махровою).
*   **`Chamomile.java`**: Підклас Ромашки. Містить специфічні поля: `coreSize` (розмір серцевини), `petalCount` (кількість пелюсток).
*   **`Lily.java`**: Підклас Лілії. Містить специфічні поля: `fragrance` (рівень аромату), `petalCount` (кількість пелюсток).
*   **`Peony.java`**: Підклас Півонії. Містить специфічні поля: `bloomStage` (стадія розкриття бутона), `isFragrant` (наявність аромату).
*   **`Orchid.java`**: Підклас Орхідеї. Містить специфічні поля: `variety` (сорт орхідеї), `isEpiphytic` (чи є епіфітною).
*   **`Accessory.java`**: Об'єкт декору для букету. Містить поля `id`, `name`, `price` та `color`.
*   **`Bouquet.java`**: Клас букету. Об'єднує списки квітів та аксесуарів. Реалізує ключові алгоритми: `calculateTotalPrice()` (розрахунок вартості), `sortFlowersByFreshness()` (сортування квітів за свіжістю) та `findFlowersByStemLength(min, max)` (пошук квітів за стеблом).

### Пакет `src/database/` (Збереження даних)
*   **`DatabaseManager.java`**: Управління базою даних SQLite через JDBC. Ініціалізує БД, створює таблиці `flowers`, `bouquets`, `accessories` та `bouquet_flowers` (багато-до-багатьох), виконує автоматичне додавання нових колонок у разі оновлення схеми та реалізує CRUD-операції з підтримкою каскадного видалення.

### Пакет `src/services/` (Координація логіки)
*   **`FlowerShop.java`**: Сервісний шар та головний координатор. Поєднує UI з базою даних, забезпечує ініціалізацію тестового каталогу з 22 квітів при першому запуску, керує транзакціями додавання, видалення та зміни об'єктів.

### Пакет `src/ui/` (Графічний інтерфейс користувача)
*   **`FlowerShopApp.java`**: Головний контейнер вікна на основі `BorderPane`. Містить бокову панель навігації, статус-бар для відображення UX-повідомлень та StackPane для динамічної зміни панелей.
*   **`CatalogPane.java`**: Панель каталогу квітів. Містить ліву панель комбінованої Live-фільтрації в реальному часі через `FilteredList` (за назвою, типами, ціною, стеблом, кольорами) та праву таблицю з функціями видалення й додавання квітів. Реалізує діалог створення квітки з динамічними полями та валідацією.
*   **`BouquetPane.java`**: Панель керування букетами. Містить пошуковий рядок для фільтрації букетів, список букетів, дві таблиці вмісту (квіти та аксесуари), діалог вибору й додавання аксесуарів, функції сортування та перейменування букетів.
*   **`SearchPane.java`**: Інтерфейс для швидкого пошуку квітів за довжиною стебла всередині обраного букету.
*   **`styles.css`**: Таблиця CSS-стилів додатка. Визначає колірну палітру на основі бірюзових та пастельних тонів, стилізує системні контроли, скролбари, діалоги, ефекти наведення та скруглення карт-контейнерів.

### Пакет `src/menu/` (Консольне меню)
*   **`Command.java`**: Функціональний інтерфейс патерну проектування "Команда".
*   **`ConsoleMenu.java`**: Реалізація текстового консольного меню програми для відлагодження через термінал.

### Конфігурація
*   **`src/resources/log4j2.xml`**: Конфігурація системи логування Log4j2. Визначає виведення в консоль (`ConsoleLogger`), збереження з ротацією в щоденні архіви у папці `logs/` (`FileLogger`) та надсилання критичних помилок на пошту `mygchinkavasil@gmail.com` через SMTP (`MailLogger`).

## Тестування проекту

Система автоматизованого тестування розділена на дві категорії:

1. **Модульні тести (`test/unit/`)**: Перевіряють бізнес-логіку методів в ізоляції.
   * `database/DatabaseManagerTest.java`: Тести операцій SQLite та каскадних транзакцій.
   * `flowers/`: Тести алгоритмів букетів (`BouquetTest.java`), конструкторів аксесуарів (`AccessoryTest.java`) та моделей квітів.
   * `services/FlowerShopTest.java`: Тести логіки сервісного координатора.
   * `menu/CommandTest.java`: Тести патерну команд.

2. **Автоматизовані GUI-тести (`test/automation/`)**:
   * `ui/FlowerShopAppTest.java`: Інтеграційні UI-тести на основі фреймворку **TestFX**. Програмно симулюють кліки користувача, навігацію, заповнення форм фільтрації й додавання елементів та перевіряють реакцію графічного інтерфейсу.
